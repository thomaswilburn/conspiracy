Conspiracy - demonstration elements

The demos on this page inherit from an element that integrates Conspiracy for rendering. On construction, each element checks to see if its class has a static template property. If it does, that property is used to create a Conspiracy instance as this.ui, and the element has a render() method that will update the Conspiracy with the element itself as the data object. You can see the complete superclass here: 


import { Conspiracy } from "../src/index.js";

export default class ConspiracyElement extends HTMLElement {

  constructor() {
    super();
    var Class = new.target;
    if (Class.template) {
      var root = this.attachShadow({ mode: "open" });
      this.ui = new Conspiracy(Class.template);
      this.ui.attach(root);
    }
  }

  render() {
    if (this.ui) this.ui.update(this);
  }
}

Because the Conspiracy template is applied to the shadow DOM, one catch is that :on.event directives should be added with the .composed option set, so that the event will pass the shadow DOM boundary and reach the element itself.

Paginated table

The <paginated-table> is a behavioral component that adds search and pagination to a table that it wraps. The source code is available here

There are two interesting things about the writing for this component. The first is the pagination drop-down. Since Conspiracy loops operate on collections, we need a way to translate a numerical range into a set of objects, ideally retaining reference identity between different list counts. To do accomplish this, the "pages" collection is actually a Proxy for an Array that generates new objects if an empty numerical index is accessed. Since setting the length of an array automatically changes its bounds (including trimming off anything past the length), we can essentially just set the length of the array and let the Proxy produce the correct entries on demand.

The other mildly interesting technique is that although all controls trigger a tableuichange event, we do need to know if the user altered the search or the page select, since the former will reset the latter. To do this, we use the dispatchedFrom property of the event created by :on.input.composed, and check this against the element references created by :element directives on both form controls. It's a simple test, but it works.
